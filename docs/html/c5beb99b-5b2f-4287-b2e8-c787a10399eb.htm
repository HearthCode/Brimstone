Walkthrough: Creating new benchmarks

[This is preliminary documentation and is subject to change.]

This walkthrough explains how to create a new benchmark for Brimstone.

Prerequisites

The BrimstoneProfiler repository contains the tools required to benchmark Brimstone applications. You must have a clone of this repository present on your local machine to run benchmarks.

To create a new benchmark

  1. Create a new function in Benchnarks/BenchmarkDefinitions.cs to implement the benchmark using the following signature:

    C#
    Copy
    public void Test_MyTestName(Game g, int it) {
  // test code here
}

    g is the Game object on which to perform the benchmark, and it is the number of iterations to perform. Note that it is up to the benchmark writer to ensure the correct number of iterations are run, eg. with a for loop.

  2. Define the starting game. By default, the game will be created by calling Default_Setup in BenchmarkDefinitions.cs before the benchmark function is called. If you wish to override this, create a new game creation function as follows:

    C#
    Copy
    public static Game Setup_MyGame() {
  // game creation code
  return myGame;
}

    Do not call GameStart in your setup function.

     Tip

    Benchmarks provides the NewEmptyGame, NewPopulatedGame and NewScenarioGame static helper methods to allow you to create games quickly. See the Benchmarks/Program.cs source code for details.

  3. Declare the benchmark. The Tests dictionary defined at the top of BenchmarkDefinitions.cs declares all of the benchmarks with the following fields:

    • Short name of the test. This is the name that can be filtered on using the --filter switch when you run benchmarks.exe.

    • Full name of the test, used in the CSV output and as a descriptive identifier of the test's purpose.

    • The method to call to execute the test.

    • [Optional] The game setup function to run before the test. If not specified, Default_Setup is used.

    • [Optional] The number of iterations to run. If not specified, this defaults to DefaultIterations defined in BenchmarkDefinitions.cs.

  4. Re-compile Benchmarks and run at the command-line as described in Walkthrough: Benchmarking a single build to view the results.
