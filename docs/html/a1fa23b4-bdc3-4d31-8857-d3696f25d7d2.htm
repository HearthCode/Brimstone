Brimstone Pre-alpha Update 3 Release Notes

[This is preliminary documentation and is subject to change.]

Brimstone Pre-alpha Update 3 is the first official release of Brimstone and is the first version to offer a complete end-to-end game flow which can be used to simulate full games using a minimal set of cards. For a full list of changes, see Pre-alpha Update 3.

Implemented game features

In this release you can perform all of the standard user actions: mulligan, play card, target, attack, end turn and concede. The follow card mechanics are implemented:

  • Taunt

  • Divine Shield

  • Stealth

  • Battlecry

  • Deathrattle

  • Damage triggers

  • Start turn and end turn triggers

  • On-play triggers

  • After-play triggers

  • On-death triggers

  • Card draw via trigger

  • Summon via trigger (partial support)

  • Overdraw

  • Fatigue

  • Hero power

Weapons, enchantments, auras, secrets, Windfury, Discover (including Tracking), Inspire, Forgetful (50% chance to attack the wrong enemy), Poisonous, healing, discard, bounce-to-hand, armor, mana gain (including The Coin) and any other mechanics not listed above are not yet implemented.

Known issues

  • The following unit tests are currently failing: TestCultMaster, TestDeathrattleVsOnDeathTriggers, TestDominantPlayerBug1, TestDominantPlayerBug2, TestEventsProcessInTimestampOrder, TestQueue1ResolvesBeforeQueue2Populates, TestTriggersProcessInTimestampOrder.

  • Multi-threaded performance with Mono is extremely slow. We recommend you use single-threaded behaviour when working with Mono. We will be transitioning the codebase to .NET Core before the first alpha release.

  • Cross-commit benchmarking is not compatible with older releases. This will be addressed in the next update.
