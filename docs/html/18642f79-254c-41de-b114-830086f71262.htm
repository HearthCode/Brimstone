Project Structure

[This is preliminary documentation and is subject to change.]

This document explains how the Brimstone project is laid out so you can find your way around.

In this section:

  • Repositories
  • Brimstone directory layout
Repositories

  • Brimstone contains the Brimstone class library, unit test suite, the Brimstone Visualizer debugging tool and the documentation web site.

  • BrimstoneProfiler contains the single-build and cross-commit benchmarking tools.

Brimstone directory layout
Repo Structure

  • Brimstone - the C# Brimstone class library project

    • Games - per-game card definitions

  • BrimstoneDocumentation - the Sandcastle/MAML documentation project

    • Content - MAML topic pages

    • media - images for use in topic pages

  • BrimstoneTests - the NUnit Framework Brimstone unit test suite

  • BrimstoneVisualizer - the WPF/XAML Brimstone game script debugging tool

  • Examples - various usage examples of Brimstone

    • VisualizerScripts - example scripts for use with BrimstoneVisualizer

  • docs - the auto-generated web site output from BrimstoneDocumentation. This will only be created when building the solution in Documentation build configuration.

  • packages - NuGet packages for the solution. These will be automatically downloaded on first build.

  • submodules - Git submodules for the solution. These will be automatically cloned when you use git clone --recursive or when you clone using the Visual Studio 2015 GitHub extension.
