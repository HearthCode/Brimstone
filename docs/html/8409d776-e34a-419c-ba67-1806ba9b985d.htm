Contributor Guide

[This is preliminary documentation and is subject to change.]

This guide is aimed at those who wish to submit code, card design and documentation contributions to Brimstone.

We aim for a high quality of code and documentation. Code submissions must follow our coding style guidelines and include unit tests where applicable. Changes which may notably impact performance must be profiled to assess whether the new code creates a bottleneck.

Code submissions must include XML documentation of all public classes, methods and properties consistent with existing documentation style and in good English. For non-native English speakers, we can provide you with help in our Discord channel.

Documentation submissions for the Brimstone web site must be written consistent with existing documentation style and in good English.
